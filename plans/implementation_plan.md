# tsinfer 1.0 Implementation Plan

## Overview

We're on branch `tsinfer-1-plans`. The existing codebase has ~9,900 lines of source and
~11,200 lines of tests, essentially all of which needs to be replaced by the new
architecture. The two things we preserve are `_tsinfermodule.c` / the compiled `.so`
(the C extension) and `tests/test_python_c.py`.

---

## Step 0: Dependencies and pyproject.toml

Before writing any code, update `pyproject.toml`:
- `zarr<3` → `zarr>=3` (Zarr v3 throughout)
- Drop `lmdb`, `sortedcontainers`, `numba` (all tied to old `formats.py` / `SampleData`)
- Add `bio2zarr` to core dependencies (or at least as optional, TBD based on how VCZ
  reading works)
- `tomllib` is stdlib since Python 3.11, no new dep needed for TOML config parsing
- The `algorithm.py` Python engine moves to tests so add no test deps for it, but confirm
  `msprime` remains in test group

## Step 2: New module structure

The new `tsinfer/` package will have these modules (built in order):

```
tsinfer/
  __init__.py       # Public API: Config, Source, infer_ancestors, match, post_process
  __main__.py       # CLI entry point (thin wrapper on cli.py)
  _version.py       # Generated by setuptools_scm
  config.py         # Config, Source, AncestralState, AncestorsConfig, MatchConfig, etc.
  vcz.py            # VCZ reading utilities (open_store, resolve_field, etc.)
  ancestors.py      # infer_ancestors — builds ancestor VCZ from sample VCZ
  matching.py       # Matcher, make_root_ts, extend_ts, compute_groups, group_by_linesweep
  pipeline.py       # match, post_process, run — the high-level pipeline loop
  cli.py            # Click/argparse CLI: infer-ancestors, match, post-process, run, config
```

## Step 3: Test helpers (build first, before any source code)

`tests/helpers.py` — the foundation everything else rests on:

```python
make_sample_vcz(genotypes, positions, alleles, ancestral_state, sequence_length, **kwargs)
make_ancestor_vcz(genotypes, positions, alleles, times, focal_positions, sequence_intervals)
ts_to_sample_vcz(ts, ancestral_state="REF") -> zarr.Group
```

These use `zarr.store.MemoryStore` — no files needed. Tests for the helpers themselves go
in `tests/test_helpers.py` to catch regressions early.

## Step 4: Config system (`config.py` + `tests/test_config.py`)

Implement the dataclasses from the design:
- `Source(path, site_mask=None, sample_mask=None, sample_time=None, name=None)`
- `AncestralState(path, field)`
- `AncestorsConfig(path, sources, max_gap_length=...)`
- `MatchConfig(sources, output, recombination_rate, mismatch_ratio=1.0,
  path_compression=True)`
- `IndividualMetadataConfig(fields, population=None)`
- `Config(ancestral_state, sources, ancestors, match, individual_metadata=None)` with
  `Config.from_toml(path)`

Tests cover:
- Direct construction
- TOML round-trip
- Field resolution rules (string, `{path, field}`, scalar)
- Path resolution relative to config file location
- Validation errors (missing required fields, bad paths, wrong types)

## Step 5: VCZ utilities (`vcz.py`)

Low-level helpers for opening and reading VCZ stores:
- `open_store(path_or_group)` — accepts path or in-memory group
- `resolve_field(store, field_spec, join_key)` — implements the three field resolution
  rules
- Array accessors for `call_genotype`, `variant_position`, `variant_alleles`, `sample_id`,
  `sample_time`, etc.

Tested implicitly through the ancestor and matching tests.

## Step 6: `infer_ancestors` (`ancestors.py` + `tests/test_ancestors.py`)

This is the most self-contained piece and a natural starting point for real logic.

**Implementation:**
1. Load sample VCZ, apply `site_mask` and `sample_mask`
2. Compute inference sites (positions passing mask with valid ancestral state)
3. Compute `sequence_intervals` from inference site positions and `max_gap_length`
4. Run the existing C ancestor generation logic (via `_tsinfer.AncestorBuilder`)
5. Apply gap clipping — clip ancestor spans to their containing interval, split any
   ancestors that span a gap boundary
6. Write the ancestor VCZ format (arrays as specified in the design)

**Tests (Tier 1 hand-constructed):** Use `make_sample_vcz` to build inputs, call
`infer_ancestors`, use the Python engine from `tests/algorithm.py` as reference oracle
for correctness. Named test cases from the design:
- Single site
- Missing flanks (left only, right only, both)
- `max_focal_positions > 1`
- Ancestor clipped by gap interval
- Site at position 0 / `sequence_length`
- All-missing site
- Site mask / sample mask exclusions
- Diploid vs haploid
- All-identical samples

## Step 7: Matching infrastructure (`matching.py` + `tests/test_matching.py`)

Three components, testable independently:

**`compute_groups(haplotypes, times, is_ancestor, start_positions, end_positions)`:**
- Returns ordered list of haplotype-index groups
- Linesweep regime for early ancestors, epoch regime for large epochs
- Tests: single epoch, multiple epochs, interleaved ancient samples, same-time ancestors
  with/without overlap

**`make_root_ts(sequence_length, positions, sequence_intervals)`:**
- Builds the empty root tree sequence with `sequence_intervals` in metadata
- Tests: verify metadata round-trip, correct sequence length

**`Matcher(ts, positions, recombination_rate, mismatch_ratio, path_compression,
num_threads)`:**
- Wraps `_tsinfer.AncestorMatcher` via `restore_tree_sequence_builder`
- `match(haplotypes) -> list[MatchResult]`
- Tests: match a single haplotype against a known ts, verify path/mutations

**`extend_ts(ts, node_times, results, node_metadata, create_individuals, ploidy,
path_compression)`:**
- Adds nodes, edges (gap-clipped), mutations, individuals
- Tests: verify node count, edge clipping at gap boundaries, individual grouping by
  ploidy, metadata encoding

## Step 8: Pipeline (`pipeline.py` + `tests/test_pipeline.py`)

Tier 2 simulation-based end-to-end tests live here.

```python
def match(cfg, reference_ts=None) -> tskit.TreeSequence
def post_process(ts, cfg) -> tskit.TreeSequence
def run(cfg) -> tskit.TreeSequence  # infer_ancestors + match + post_process
```

Tests use `ts_to_sample_vcz(msprime_ts)` as input, run the full pipeline, and verify:
- Output is a valid tree sequence
- All samples appear in output
- Parametrised over sample count, ploidy, sequence length, mut/rec rates
- Ancient samples interleaved correctly (time ordering in output)

---

## Step 9: CLI (`cli.py` + `tests/test_cli.py`)

Thin wrappers calling pipeline functions. Recommend `click` (already commonly used in
the tskit ecosystem).

Commands: `infer-ancestors`, `match`, `post-process`, `run`, `config show`,
`config check`

Flags: `--threads N`, `--force`, `--progress`, `-v/--verbose`

Tests: Tier 3 — construct `Config` programmatically with in-memory stores, invoke
commands via `click.testing.CliRunner`. TOML-specific tests (path resolution, invalid
values) use `tmp_path`.

---

## Step 10: Node and individual metadata

This spans ancestors and matching. Implement after both are working:
- Node metadata schema set automatically on `make_root_ts`
- `source`, `sample_id`, `ploidy_index` populated in `extend_ts`
- Individual metadata from `[individual_metadata]` field mappings
- Population assignment from `population` field
